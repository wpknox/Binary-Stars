Stellar-Evolution-Clustering | User Manual

Home Page

The home page is the entry point for the application. The home page contains a list of saved user queries. After submitting a query, it is automatically saved to this list.

Query Form

1. The first step when creating a query is selecting the database. Two databases are implemented for this version, COSMIC and POSYDON.

2. The second step is to select the attributes to cluster with. At least one attribute is required.

3. Third, attributes can be weighted. The larger the weight of an attribute, the more clustering will consider it. The sum of weights must add to 100. Selecting “Allow empty inputs” will evenly weight all attributes.

4. Fourth, the user can select which clustering algorithm to use. The two algorithms, K-Means and DBScan, each have their own pros and cons. More detail is located in the About Page.

5. Fifth, based on the clustering algorithm selected, users may need to provide additional parameters. For example, for K-Means users need to specify the number of clusters.

6. Lastly, users can review all input into the form. When satisfied, the query can be submitted to the backend.

Cluster Visualization

After a query submission, users can visualize their data with 2D and 3D graphs. The default graph on query submission is the two-attribute display over time.

The user can select different combinations of attributes to display using the configure button. The type of graph can also change. The following are possible graphs: One attribute over time on a 2D graph, two attributes over time on a 3D graph, or three attributes at single time steps on a 3D graph.
